Comparison paragraph:

	Overall, the replacement algorithm's in order of worst to best hit rate is: LRU->FIFO->Clock->Opt. Optimal being 
the best because it's algorithm evictes the latest page that will be used again. As a result, the miss count of the
Opt algorithm is the smallest of the 4 sorting algorithms. Clock is the next best because while it doesn't check 
for the page with the furthest reference time, it does at least scan through all the cached pages to find one that
wasn't recently used or if all were recently used, the first one that it checked. This is more efficent that FIFO
and LRU as evident by the lowewr miss count for simulations using clock. For some reason, our LRU is worse than FIFO
when it should be the other way around. LRU should have approximately the same performance as Clock since they both
work in a similar fashion; clock replaces the first oldest page it finds while LRU evicts the oldest page in the 
cache. FIFO should be the worst because it doesn't check for anything, it just replaces pages in the order that they
entered.

LRU description:

	By looking at all the simulations done using the LRU replacement algorithm, it is clear that a positive 
correlation exists between memeory size and hit rate for LRU. In other words, as the memory size for LRU increases,
the hit rate of LRU also increases. This is because LRU heavily relies on memeory size in order to function optimally.
The LRU replacement algorithm involves searching through all the stored pages and replacing the one that was least
recently used. The larger the memory size, the more pages that can be stored. If there are more pages stored, there
are more options to choose from and thus the most optimal page can replaced.
